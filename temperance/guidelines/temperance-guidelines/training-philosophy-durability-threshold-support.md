# Philosophy Overlay — Durability-First / Threshold-Forward / Support-Modality Positive

Status: reusable overlay / default.
Profile type: philosophy.
Profile id: durability-first-threshold-forward-support-positive.

## Objective and main limiter

- Preserve a large aerobic engine while progressing specific load without repeating density-driven injury patterns.
- The main limiter this philosophy is trying to manage is hidden mechanical and threshold density.

## Primary specificity axis

- Within the event overlay's demands, favor threshold / strong aerobic work and durable specific work over frequent sharp intensity.
- Treat support modalities as active planning tools, not passive backup.

## Hard-session priority order

- preserve the event overlay's top specific anchor first
- threshold / strong aerobic hard as the preferred non-anchor quality
- event-specific hard when it improves race relevance without breaking structure
- sharp hard as supporting only

## Spacing / density preferences

- Use 3-day spacing as the preferred local structure when it fits the block.
- Use a rolling 9-day lens to review density, subtype clustering, and accumulated stress.
- Hard support-modality work is acceptable when the block still contains real compensation and the main mechanical stress remains protected.
- As a planning heuristic, each 3-day cycle should average roughly 40-45% of the target weekly total_load. Cycles that drift repeatedly above that range are a soft alert for hidden stacking even if individual sessions look reasonable in isolation.

## Progression priorities

- Keep total load productive through support modalities when specific load must stay capped.
- During Base / Capacity Build, widen the range rather than living in one middle lane: lots of easy / aerobic support, plus occasional ceiling-touch work that still fits the weekly budget.
- Progress specific load and long-duration structure before adding extra sharpness.
- Use doubles when they improve absorption, spacing, or anchor-hitting efficiency.

## Alert sensitivity / default responses

- Watch threshold density, hidden moderate drift, and extra weekend stacking closely.
- Default response: hold progression first, reduce the next specific-load target modestly if needed, and substitute support-modality work before forcing another hard run.

## Explicit non-goals

- Do not make Norwegian-style double threshold the default pattern.
- Do not let the whole week flatten into hidden moderate work.
- Do not let support work turn into empty filler.
- Do not make sharp intensity the dominant organizing principle.
