## Preferred Strategic Direction Going Forward

- Use a **3-day cycle** approach instead

### 3-Day Cycle Idea

Working interpretation for future planning:

The 3-day cycle is **not** intended as a rigid repeating easy / moderate / hard sequence.

Instead, it is mainly a spacing principle:
- aim for roughly **3 days between the hardest run-stress days** when possible
- avoid concentrating meaningful run stress too tightly
- use cross-training and lower-impact days flexibly between hard run sessions

A hard run-stress day can be:
- a long run
- a marathon-specific workout
- a threshold session
- another meaningful run session with real mechanical cost

The intervening days do not need to follow a fixed script. Depending on context, they can include:
- cross-training support
- easy running
- lower-load aerobic work
- recovery-oriented sessions
- or rest

Important planning exception:
- if a hard session is initially set for **Friday**, it may be smarter to move it to **Thursday** and use **Friday as rest or very low load**
- this still follows the core idea of better spacing the hardest days and protecting durability

### Why this idea fits Matt better

Compared with classic double-threshold structures, the 3-day cycle can:
- preserve high total aerobic load
- keep rTSS progression moving upward
- use cross-training strategically to support fitness
- reduce the density of hard mechanical run stress without enforcing a rigid template
- make long-run progression easier to integrate
- reduce the risk of repeating the pattern of too much threshold density plus extra weekend loading

### Guardrails for using the 3-day cycle

- Cross-training should support the plan, not become an excuse to stack hidden fatigue everywhere
- The hardest run sessions should still be intentionally chosen and limited
- Long-run progression remains a major pillar and should be protected
- Mechanical stress should rise progressively, even if total TSS rises faster through cross-training support
- Durability is the target outcome, not just big numbers on paper
- Do not automatically panic if an in-between day includes a hard x-train session, as long as the cycle still includes at least one easy or rest / very low load day to compensate
- A useful planning heuristic is for each 3-day cycle to average roughly **40-45% of target weekly load**; if cycles repeatedly drift too far from that, it should be treated as an alert signal rather than as a hard rule violation

## Core Interpretation Rules

When analyzing training or making progression rules for Matt:

1. **Do not anchor only on the recent rehab block.**
   Matt's true baseline is that of a historically high-volume runner.

2. **Do not confuse aerobic tolerance with mechanical tolerance.**
   He may handle very large total cardio load while still needing caution with current run-specific load.

3. **Respect his ability to handle substantial load.**
   Moderate-to-high overall load is not inherently too much for him.

4. **Be cautious with the patterns that appear injury-linked.**
   Especially:
   - too much double-threshold density
   - extra weekend load on top of hard quality structure
   - oversized long-run / mega-day experiments
   - rapid increases in run-specific load during return-to-run

5. **Use rTSS and long-run progression as the main organizing principles.**
   Weekly build quality matters more than Garmin readiness scores.

6. **Prefer sustainable progression over heroic spikes.**
   Matt has the engine for big work, but the coaching task is to channel that capacity without recreating the same injury dynamics.

7. **Use cross-training strategically, not just as filler.**
   It should help maintain or build aerobic load while protecting current mechanical capacity.
