# Event Overlay — Marathon Default

Status: reusable overlay / default.
Profile type: event.
Profile id: marathon-default.

## Objective and main limiter

- Build the highest usable marathon fitness the current body can absorb.
- The main limiter is usually durable absorption of large running load and long-duration specific stress.

## Primary specificity axis

- Running load is the primary specific load.
- The key duration anchor is the long run.
- `Specific hard` means marathon-oriented work that improves fatigue resistance, race-pace durability, and long-duration economy.

Phase note: in Return / Re-entry and Base / Capacity Build phases, specific hard should mean durable long-run work and progression runs rather than targeted race-pace sessions. True marathon-pace specificity is most productive closer to the event. Do not force it early when the primary need is base durability and running-load absorption.

## Hard-session priority order

- long-duration hard
- specific hard
- threshold hard
- sharp hard

## Spacing / density preferences

- Protect the long run as the main mechanical anchor of the week or block.
- Keep meaningful run-stress days spaced rather than stacked.
- Use support modalities to preserve total load when running cannot yet carry enough of it.

## Progression priorities

- Grow weekly running load coherently.
- Stage long-run progression and long-run quality.
- Increase marathon specificity as the event approaches without breaking the rest of the structure.

## Alert sensitivity / default responses

- Be especially alert to running-share jumps, oversized long-run increases, and weekend mechanical stacking.
- Default response: freeze long-run or running-load progression before sacrificing overall weekly structure.

## Explicit non-goals

- Do not treat total load alone as proof of marathon readiness.
- Do not push marathon-specific work early just because the engine can support it.
- Do not let sharp intensity become the dominant form of hard work.

## Good week notes

- A good week includes a coherent long run, intentional hard-session types, appropriate running share, and a finish state that still supports the next week.
- Marathon specificity means durable race-relevant work, not just faster intervals.
